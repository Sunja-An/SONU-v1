## 요구사항

---

<aside>
⭐

> 참여할 수 있는 사람들
> 
1. 20명 이상의 사람들이 대회에 참여해서 5명씩 팀을 짤 수 있어야 됨.
2. 경매 형태로 진행하여서, 팀장 4명-6명이 Real Time 으로 접속하여서, 실시간으로 경매를 할 수 있어야 됨.
3. 사람들은 자기소개, 디스코드 프로필, 디스코드 이름, 역할군, 잘 사용하는 캐릭터 3개에 대해서 표기할 수 있어야 됨.
    1. 자기소개 ( 50줄 이내 )
    2. 디스코드 프로필
    3. 디스코드 이름
    4. 역할군 ( 감시자, 타격대, 연막, 척후대 )
    5. 잘 사용하는 캐릭터 3개
4. 팀장으로써 참여하는지, 팀원으로써 참여하는지에 대한 구분을 할 수 있어야 됨.
</aside>

<aside>
⭐

> 팀장의 권한
> 
1. 경매에 직접 참여할 수 있는 시스템을 가짐.
    1. 총 1000포인트로 시작한다.
2. 경매에 직접 참여할 수 있는 사람들에게는 특별한 URL 을 전달
    1. 실제 경매가 이루어지는 URL 을 전달해서 경매가 이루어질 수 있도록 함.
3. 경매에 대해서 실제 Point 들에 대해서 교환이 이루어질 수 있도록 함.
4. 만약 경매가 이루어지고, 사람들을 GET 했을 때, 해당 사람들이 팀원에게 들어갈 수 있도록 설정할 수 있는 Websocket 을 다룰 수 있어야 함.
</aside>

<aside>
⭐

> 팀원의 권한
> 
1. 본인이 경매에 들어갔을 때, 경매에 본인의 이력서 ( 뽐내고 싶은 것들 ) 에 대해 설명할 수 있는 란이 있어야 됨.
</aside>

<aside>
⭐

> 운영자의 권한
> 
1. 대회를 개최한 운영자들에 대해서는 대회에 들어오고 싶어하는 사람들에 대해서 초대들의 Accept, Reject 를 할 수 있음.
2. 경매를 시작할 수 있는 시간대를 설정할 수 있음.
    1. 이 부분 개발 잘해야될 것 같은데,,
3. 실제로 들어가있는 사람들에 대해 포인트들에 대해 환산을 시킬 수 있음.
</aside>

> 경매 시스템 로직
> 
1. 경매에 시작하기 전, 각 경매 입찰자들에게 WebSocket 링크 제시
2. Websocket 링크 받은 후, 들어왔을 때, 신경써야되는 Event들
3. 운영자측이 대회를 시작하게 되면, 팀원들과 팀장들의 Shuffle 이 시작됨
    1. 팀원들은 자신이 어느 순번대로 나갈 것인가?
    2. 팀장들은 몇 번째 팀인가?
4. Shuffle 이 끝나고 나면, 팀장이 팀원들에 대해서 입찰을 시작할 수 있음
5. 입찰을 시작하고, 만약 당첨이 된다하면, 입찰된 팀쪽으로 해당 팀원이 들어갔음을 UI/UX 상으로 표기할 수 있어야됨
    1. 또한 입찰되었다는 마크를 붙일 수 있어야 한다.
    2. 또한 유찰되었다는 마크를 붙일 수 있어야 한다.
        1. 이 부분의 디자인은 AI 에게 맡김
6. 만약 입찰이 1바퀴를 다 돌고 난 후, 금액이 남아있다면 유찰들에 대해서 금액을 투자할 수 있도록 한다.
7. 마지막 1명이 팀원이 남게되면 자동으로 해당 팀원에게 들어갈 수 있다.
